Geohash Polygon Query Pipeline

"Find all 7-Elevens inside this polygon" — from API call to final result

1
User Draws a Polygon on the Map
The user draws an irregular polygon on the map. The frontend sends the polygon vertices to GeoServer.
2
Compute Bounding Box
Loop through all vertices. Track the smallest and largest lon/lat values. The bbox is the rectangle that fully encloses the polygon.
3
Pick Geohash Precision from Bbox Size
Measure the bbox span in miles. Pick precision to match available indexes.
lat_span = 5.52 miles, lon_span = 3.66 miles → max = 5.52 miles 5.52 > 20? No (skip precision 5) 5.52 > 1? Yes → Precision 6 (cell ≈ 0.61 miles, uses ix_mapdata4)
4
Walk the Geohash Grid (SW → NE)
Encode the SW corner to get the starting cell. Walk East and North until the bbox is fully covered. Result: ~35 geohash-6 cells.
5
DocDB Index Scan (B-Tree on geohash prefix)
YSQL sends ~35 exact-match lookups to DocDB. The md_name = '7-Eleven' filter is pushed down to the storage layer.
WHERE LEFT(geo_hash10, 6) IN ('9x1y8k','9x1y8m', ... ~35 cells ...) AND md_name = '7-Eleven' → Returns 18 candidate rows (7-Elevens in the bbox area)
6
Polygon Filter (Ray-Casting Point-in-Polygon)
For each candidate row, cast a ray eastward and count polygon edge crossings. Odd = inside. Even = outside. Removes false positives from bbox corners.
7
Final Result → 14 rows
18 candidates → 4 discarded (bbox corners outside polygon) → 14 7-Elevens truly inside the polygon returned to the application.

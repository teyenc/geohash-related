Geohash Polygon Query Pipeline

"Find all 7-Elevens inside this polygon" — from API call to final result

1
User Draws a Polygon on the Map
The user draws an irregular polygon on the map (e.g., a neighborhood boundary). The frontend sends the polygon vertices to GeoServer / the application.
Polygon Vertices: (-111.95, 40.72) ← SW corner (-111.88, 40.72) ← SE corner (-111.88, 40.78) ← NE-ish (-111.92, 40.80) ← N peak (-111.95, 40.78) ← NW corner
2
Compute Bounding Box (ST_XMin/XMax/YMin/YMax)
Loop through all 5 vertices. Track the smallest and largest lon/lat values.
Vertex 1: lon=-111.95 lat=40.72 min_lon=-111.95 max_lon=-111.95 min_lat=40.72 max_lat=40.72 Vertex 2: lon=-111.88 lat=40.72 max_lon updated → -111.88 Vertex 3: lon=-111.88 lat=40.78 max_lat updated → 40.78 Vertex 4: lon=-111.92 lat=40.80 max_lat updated → 40.80 Vertex 5: lon=-111.95 lat=40.78 (no changes) Bounding Box: West = -111.95 South = 40.72 East = -111.88 North = 40.80
3
geohash_cells_for_bbox — Pick Precision
Measure the bbox span in miles. Pick precision to match available indexes.
lat_span = (40.80 - 40.72) * 69 = 5.52 miles lon_span = (111.95 - 111.88) * 69 * cos(40.76°) = 3.66 miles max_span = 5.52 miles 5.52 > 20? No (skip precision 5) 5.52 > 1? Yes → Precision 6 (uses ix_mapdata4) Cell size at precision 6: ~0.61 miles
4
geohash_cells_for_bbox — Walk the Grid
Encode the SW corner to get the starting cell. Then walk East and North until the bbox is fully covered.
SW corner: geohash_encode(40.72, -111.95, 6) = '9x1y8k' Row 1 (South): Start: 9x1y8k → step E → 9x1y8m → step E → 9x1y8q → ... Stop when cell's west edge >= -111.88 (East boundary) Row 2: Start: geohash_adjacent('9x1y8k', 'n') = '9x1y8s' Walk east again: 9x1y8s → 9x1y8t → 9x1y8u → ... ... repeat until row's south edge >= 40.80 (North boundary) ...
Result: ~35 geohash-6 cells covering the bbox '9x1y8k','9x1y8m','9x1y8q','9x1y8s','9x1y8t', ...
5
DocDB Index Scan (ix_mapdata4) + Name Filter
YSQL sends ~35 exact-match lookups to DocDB on the ix_mapdata4 index. The md_name = '7-Eleven' filter is pushed down to the storage layer.
WHERE LEFT(geo_hash10, 6) IN ('9x1y8k','9x1y8m', ... ~35 cells ...) AND md_name = '7-Eleven' DocDB scans LSM index across TServers in parallel. Discards non-7-Eleven rows at storage layer. Returns: 18 rows (7-Elevens in the bbox area)
6
YSQL Polygon Filter (ST_Contains → Ray-Casting)
For each of the 18 rows, YSQL runs point_in_polygon(). It casts a ray eastward from the point and counts how many polygon edges it crosses. Odd = inside. Even = outside.
ST_Contains(polygon, point) → calls point_in_polygon() Row 1: 7-Eleven on State St → ray crosses 1 edge → INSIDE Row 2: 7-Eleven on Main St → ray crosses 1 edge → INSIDE Row 3: 7-Eleven on Center Ave → ray crosses 1 edge → INSIDE ... Row 14: 7-Eleven on Highland Dr → ray crosses 1 edge → INSIDE Row 15: 7-Eleven on Park Blvd → ray crosses 0 edges → OUTSIDE (bbox corner) Row 16: 7-Eleven on Canyon Rd → ray crosses 0 edges → OUTSIDE (bbox corner) Row 17: 7-Eleven on Ridge Way → ray crosses 2 edges → OUTSIDE (concave notch) Row 18: 7-Eleven on Valley Dr → ray crosses 0 edges → OUTSIDE (bbox corner)
7
Final Result → Application / GeoServer
14 rows returned. Only 7-Elevens truly inside the polygon.
